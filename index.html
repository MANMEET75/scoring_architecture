Data sources & feature derivation
Research → Development ▶
Raw data inputs
Clickstream data
Page-level click events per user — which pages visited, what was clicked, time on each page, device type. Captures passive browsing intent.
User behaviour
GA events & sessions
Google Analytics session data — session count, dwell time, bounce rate, entry/exit pages, funnel progression events across sell journey.
Session signals
Transaction data
Historical transaction records — sell completions, inspection bookings, quote requests, offer acceptances. Ground truth for conversion history.
Conversion history
CRM interaction history
WhatsApp delivered/read/replied, SMS open/click rates, past campaign response — per-channel conversion history per user.
Channel response
Form & disposition data
Quote form submissions, lead form fills, logged-in activity, disposition outcomes from call centre (interested / not interested / callback).
Intent signals
Feature derivation approach: Raw events were aggregated into behavioural features at different time windows — last 7 days, last 30 days, last 90 days. Recency-weighted aggregations were used so recent activity counted more than older activity. Features were derived in-house from raw data, not pulled from CleverTap, establishing full internal ownership of the scoring pipeline.

02
The 6 scoring dimensions — design & formulas
Core Design ▶
All scores are normalised to 0–100 using weighted component aggregation. Higher score = higher intent / engagement. Each score is independently computable — they can be used individually or combined into a composite score for ranking users.
Engagement score
Measures how actively a seller is engaging with the Cars24 platform — frequency, recency, and depth of digital interactions.
Components:
· Session frequency (30-day window)
· Dwell time per session
· Page click count
· Core feature page visits (quote, sell, valuation)
Passive behaviour
Activity depth score
Measures how deep into the sell funnel a seller has progressed — exploration breadth and multi-channel touchpoints.
Components:
· Funnel stage progression (lead → inspection → offer)
· Unique pages explored per session
· Multi-channel interaction count
· Return visit depth
Funnel depth
Intent score
Measures explicit intent signals — actions that require deliberate effort from the seller, indicating genuine interest in selling.
Components:
· Quote form submissions
· Logged-in session ratio
· Lead form completions
· Disposition quality (call centre outcomes)
· Offer page views
Active intent
RFM scores
Classic recency-frequency-monetary framework extended with a depth dimension specific to the Cars24 sell journey.
Components:
· R: days since last platform activity
· F: visit / interaction frequency
· M: monetary value of past transactions
· D: depth of funnel engagement
RFM + depth
Velocity score
Captures the trend direction of a seller's activity — are they becoming more engaged (accelerating) or less engaged (decelerating)?
Components:
· 7-day vs 30-day activity ratio
· Growth / decline trend in session count
· Engagement acceleration metric
· Recent spike detection
Trend direction
Channel conversion score
Per-channel historical responsiveness — how likely is this specific seller to respond to WhatsApp vs SMS based on past behaviour?
Components:
· WhatsApp: delivered → read → replied rate
· SMS: delivery → click rate
· Past campaign conversion history
· Optimal send time signal
Channel-specific
Score normalisation approach
# Each raw score computed from weighted component sum
raw_score = w₁·component_1 + w₂·component_2 + w₃·component_3 + ...

# Normalised to 0–100 using min-max normalisation
score_normalised = ((raw_score - min_score) / (max_score - min_score)) × 100

# Weights determined from historical campaign conversion analysis
# Higher weight → component more predictive of eventual conversion

# Example: Engagement score
engagement = 0.35·session_freq + 0.25·dwell_time + 0.25·page_clicks + 0.15·core_page_visits

03
User segmentation & suppression logic
Implementation ▶
Score-based targeting tiers
High intent
Score 70–100 · Target with priority messaging · Personalised offers · All channels enabled
Medium intent
Score 40–69 · Target with standard cadence · Nurture campaigns · Selective channel use
Low intent
Score 0–39 · Suppressed from campaigns · ~40% of user base falls here · Saves cost
Suppression logic
Score threshold suppression
Users with composite score below threshold are excluded from campaign audience before message send — not filtered post-send, which wastes cost.
Pre-send filter
~40% user suppression
In a typical WhatsApp campaign, approximately 40% of the target audience is suppressed based on low intent scores — dramatically reducing per-campaign cost.
40% suppressed
Channel-specific suppression
Channel conversion score enables channel-level suppression — a user with low WhatsApp responsiveness but high SMS responsiveness is routed accordingly.
Per-channel
Key design decision — suppress before send, not after: The suppression list is generated before the campaign audience is pushed to WhatsApp or SMS vendors. This ensures the cost is never incurred for low-intent users — the message is never sent, not delivered-and-ignored. This is the mechanism that directly drives the cost saving.

04
Score validation & backtesting
Validation ▶
Validation approach
Historical campaign backtestScores were computed retrospectively on historical campaign data. Verified that suppressed users (those who would have been scored low) did not convert in the historical period — confirming the suppression would not have missed conversions.
Suppression vs conversion checkCritical validation: confirmed that ~40% suppressed users had near-zero historical conversion rates. This justified suppression — the cost saving does not come at the expense of missed sales.
Score distribution sanityVerified score distributions were not degenerate — no score clustering at 0 or 100. Checked that high scorers had meaningfully higher conversion rates than medium and low scorers.
Live campaign monitoringAfter initial deployment, ran live campaigns using the scoring framework. Observed improved conversion rates on targeted audiences and confirmed cost reduction on WhatsApp sends.
The key validation question interviewers will ask: "How did you make sure suppressing 40% of users didn't reduce your conversions?" Answer: Backtested on historical data — users scored low had near-zero conversion in past campaigns. Their suppression removes cost without removing conversion. This was verified before any live suppression was applied.

06
WhatsApp communication optimisation
Cost Optimisation ▶
Per-message cost
WhatsApp Business API charges approximately ₹0.35 per outbound message. At the scale of Cars24's seller base, this accumulates rapidly without intelligent filtering.
₹0.35 / message
Volume reduction
By suppressing ~40% low-intent users, approximately 3 Cr WhatsApp sends were eliminated — users who historically never converted from WhatsApp campaigns.
~3 Cr sends cut
Annual saving
~3 Cr sends × ₹0.35 per message = ~₹1 Cr saved per year on WhatsApp sends alone — users who never responded to these messages historically.
~₹1 Cr/year
Channel conversion score enables channel-level routing: Beyond suppression, the channel conversion score routes users to their most responsive channel. A user who never responds to WhatsApp but clicks SMS links is suppressed from WhatsApp sends and targeted via SMS instead. This improves both cost efficiency and conversion rates simultaneously.

08
How this differs from the seller propensity model
Interview Q&A ▶
Both projects use similar raw inputs (clickstream, GA, CRM interactions) — the difference is what we do with those features and what problem we're solving.
Dimension Seller propensity model CRM scoring framework
Core question Will this inactive user convert in the next 30–90 days? Is this user worth contacting today, and on which channel?
Methodology Supervised ML — XGBoost trained on 1 year of labelled conversion data Formula-based — explicit weighted aggregations, no training data needed
What features do Input to a model that learns the relationship between features and conversion probability Directly become interpretable scores — session freq = engagement score, recency = RFM-R
Output Calibrated probability score (0–1) + decile ranking → micro-cohort targeting list 6 normalised scores (0–100) + tier assignment → pre-send suppression list
User base Inactive users (30+ days) — medium to long-term targeting horizon All sellers — evaluated fresh before every campaign send
Primary goal Maximise conversions by finding high-potential users in a cold base Minimise cost by suppressing users unlikely to respond right now
Explainability Partial — SHAP values can explain, but XGBoost is not fully transparent Full — CRM team can see exactly why a user has a high or low score
In production PMML model served via REST API on EC2 with CI/CD, PSI/CSI monitoring Score table refreshed daily/weekly, queried directly by CRM team
How they work together: The propensity model identifies who could convert over the next 30–90 days — a strategic targeting pool from the inactive base. The CRM scoring framework then filters that list using current engagement state — so budget isn't spent on a user the propensity model flagged as high-potential but who hasn't been active in 3 weeks. One gives the strategic pool, the other gives the operational precision.
