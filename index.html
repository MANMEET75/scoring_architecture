How this differs from the seller propensity model
Both use the same raw inputs (clickstream, GA, CRM). The difference is what we do with those features and what problem we're solving.
DimensionSeller propensity modelCRM scoring framework
Core questionWill this inactive user convert in next 30–90 days?Is this user worth contacting today, on which channel?
MethodologySupervised ML — XGBoost trained on 1 year labelled dataFormula-based — explicit weighted aggregations, no training needed
Features roleInput to a model that learns the conversion relationshipDirectly become the score — session freq = engagement, recency = RFM-R
OutputCalibrated probability (0–1) + decile → micro-cohort list6 normalised scores (0–100) + tier → pre-send suppression list
User baseInactive users (30+ days) — medium/long horizonAll sellers — evaluated fresh before every campaign send
Primary goalMaximise conversions from a cold inactive baseMinimise cost by suppressing low-intent users right now
ExplainabilityPartial — SHAP needed to explain XGBoost decisionsFull — CRM team sees exactly why score is high or low
How they work together: Propensity model identifies who could convert over 30–90 days (strategic pool from inactive base). CRM scores filter that list by current engagement state — so budget isn't spent on someone flagged high-potential but inactive for 3 weeks. One gives the strategic pool, the other gives operational precision.
